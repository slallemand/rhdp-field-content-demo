== Questions & Réponses

[.tile]
.Know
****
**Questions fréquentes à anticiper:**

**Q: Puis-je utiliser mes propres images de conteneurs ?**

A: Oui ! Ajoutez-les dans votre Helm chart. RHDP déploie tout ce qui est dans votre repository. Assurez-vous que vos images sont sur un registry accessible (quay.io, DockerHub public, ou registry interne accessible).

**Q: Que se passe-t-il si mon déploiement échoue ?**

A: ArgoCD affiche les erreurs dans son UI. Les causes courantes : images inexistantes, RBAC insuffisant, ou label `demo.redhat.com/application: "field-content"` manquant. Vérifiez les logs ArgoCD et les events Kubernetes.

**Q: Puis-je utiliser Ansible au lieu de Helm ?**

A: Absolument ! Le role `ocp4_workload_field_content` supporte les deux. Helm est recommandé pour sa simplicité, mais Ansible offre plus de flexibilité pour les workflows complexes.

**Q: Comment gérer les secrets (mots de passe, API keys) ?**

A: Ne les committez JAMAIS dans Git. Utilisez des Secrets Kubernetes créés dynamiquement via Ansible, ou des External Secrets Operator. RHDP peut injecter des secrets pré-provisionnés si configurés dans AgnosticD.

**Q: Combien de temps prend le provisionnement CNV ?**

A: Généralement 3-5 minutes pour un cluster simple. Les clusters avec beaucoup de workers ou des configurations complexes peuvent prendre 10-15 minutes.
