# Deploying Showroom Demo on RHDP

This document explains how the Showroom demo content is deployed automatically via RHDP Field Content CI.

## Overview

The Showroom demo content (`showroom-content/`) is deployed using an **App of Apps** pattern:

```
RHDP Field Content CI
  │
  └─► ArgoCD Application (root chart)
        │
        ├─► Operator Subscriptions (OpenShift Pipelines, Web Terminal)
        ├─► Hello Node.js App
        └─► Showroom Demo ◄── New!
              │
              └─► Helm Chart: components/showroom-demo/
                    ├─► Deployment (with git-clone init container)
                    ├─► Service
                    ├─► Route
                    └─► Secret (user credentials)
```

## Official RHPDS Showroom Deployer

This deployment uses the **official RHPDS Showroom deployer Helm chart** from `https://rhpds.github.io/showroom-deployer`.

The chart automatically:
- Builds the Antora site from Git repository
- Creates BuildConfig, ImageStream, Deployment, Service, Route
- Handles all OpenShift-specific configuration

## How It Works

### 1. RHDP Creates ArgoCD Application

When you order **Field Content CI** from the RHDP catalog:

```yaml
# RHDP automatically creates this ArgoCD Application
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: field-content-<guid>
  namespace: openshift-gitops
spec:
  source:
    repoURL: https://github.com/slallemand/rhdp-field-content-demo.git
    targetRevision: HEAD
    path: .  # Root of the repository
    helm:
      values: |
        deployer:
          apiUrl: "https://api.cluster-xyz.example.com:6443"
          domain: "apps.cluster-xyz.example.com"
        userInfo:
          user: admin
          password: <auto-generated>
          guid: abc123
```

### 2. Root Chart Creates Showroom Application

The root chart (`Chart.yaml` + `values.yaml` + `templates/`) contains:

**values.yaml (helmCharts section):**
```yaml
helmCharts:
  showroom-demo:
    enabled: true
    namespace: showroom
    wave: 20
    source:
      # Official RHPDS Showroom deployer Helm chart
      repoURL: https://rhpds.github.io/showroom-deployer
      chart: showroom
      targetRevision: 0.4.9
    valuesTemplate: |
      deploy:
        name: rhdp-demo
        url: {{ .Values.gitUrl | quote }}
        ref: {{ .Values.gitRef | quote }}
        path: showroom-content
```

**templates/helm-charts.yaml** automatically generates the ArgoCD Application from this configuration.

### 3. Showroom Helm Chart Deploys the Content

The Showroom chart (`components/showroom-demo/`) creates:

**Deployment with Init Container:**
```yaml
initContainers:
- name: git-clone
  image: alpine/git
  command:
  - sh
  - -c
  - |
    git clone {{ .Values.content.gitUrl }} /tmp/repo
    cd /tmp/repo
    git checkout {{ .Values.content.gitRef }}
    cp -r showroom-content/* /content/
  volumeMounts:
  - name: content
    mountPath: /content
```

**Main Container:**
```yaml
containers:
- name: showroom
  image: quay.io/rhpds/showroom:0.1.9
  env:
  - name: OPENSHIFT_CONSOLE_URL
    value: "https://console-openshift-console.apps.{{ .Values.deployer.domain }}"
  - name: USER
    value: "{{ .Values.userInfo.user }}"
  volumeMounts:
  - name: content
    mountPath: /app/content
```

**Route:**
```yaml
apiVersion: route.openshift.io/v1
kind: Route
spec:
  host: showroom-demo-field-content-demo.apps.{{ .Values.deployer.domain }}
  to:
    kind: Service
    name: showroom-demo
  tls:
    termination: edge
```

## Testing Locally

### Test the Showroom Helm Chart

```bash
cd components/showroom-demo

# Lint the chart
helm lint .

# Render templates
helm template showroom-demo . \
  --set deployer.domain=apps.cluster-abc123.example.com \
  --set deployer.apiUrl=https://api.cluster-abc123.example.com:6443 \
  --set userInfo.password=openshift123

# Deploy to a test cluster
helm install showroom-demo . \
  --create-namespace \
  --namespace field-content-demo \
  --set deployer.domain=$(oc get ingresses.config cluster -o jsonpath='{.spec.domain}') \
  --set deployer.apiUrl=$(oc whoami --show-server) \
  --set userInfo.password=$(oc whoami -t)
```

### Test the Root Chart

```bash
# From repository root
helm lint .

# Render the complete App of Apps structure
helm template field-content . \
  --set deployer.domain=apps.cluster-abc123.example.com \
  --set deployer.apiUrl=https://api.cluster-abc123.example.com:6443

# Check that showroom-demo Application is created
helm template field-content . \
  --set deployer.domain=apps.cluster-abc123.example.com | \
  grep -A 50 "name: showroom-demo"
```

## GitOps Flow

When you commit changes to this repository:

1. **Commit to GitHub:**
   ```bash
   git add showroom-content/
   git commit -m "Update demo content"
   git push origin main
   ```

2. **ArgoCD Detects Change:**
   - ArgoCD polls the repository every 3 minutes (default)
   - Detects changes in `components/showroom-demo/` or `showroom-content/`

3. **Auto-Sync Triggered:**
   - ArgoCD creates new pods
   - Init container clones the latest code
   - Showroom serves updated content

4. **Self-Healing:**
   - If you manually delete resources, ArgoCD recreates them
   - If you manually edit resources, ArgoCD reverts to Git state

## Troubleshooting

### Init Container Fails

**Symptom:** Pod stuck in `Init:Error` or `Init:CrashLoopBackOff`

**Check logs:**
```bash
oc logs -n field-content-demo deployment/showroom-demo -c git-clone
```

**Common causes:**
- Git URL unreachable → Check firewall/proxy
- Branch doesn't exist → Verify `gitRef` in values
- Insufficient permissions → Check serviceAccount

### Content Not Updating

**Force ArgoCD sync:**
```bash
argocd app sync showroom-demo
```

**Or delete the pod:**
```bash
oc delete pod -n field-content-demo -l app.kubernetes.io/name=showroom-demo
```

### Route Returns 503

**Check service endpoints:**
```bash
oc get endpoints showroom-demo -n field-content-demo
```

**Check pod readiness:**
```bash
oc get pods -n field-content-demo
```

## Cleanup

### Delete Showroom Only

```bash
oc delete application showroom-demo -n openshift-gitops
```

### Delete Everything (Full Field Content)

```bash
# This deletes the entire RHDP Field Content deployment
oc delete application field-content-<guid> -n openshift-gitops
```

Or via RHDP catalog: **My Services → <your service> → Delete**

## Required RHDP Label

All resources **must** include this label:

```yaml
metadata:
  labels:
    demo.redhat.com/application: "field-content"
```

This is already configured in:
- Root chart: `templates/showroom-demo-application.yaml`
- Showroom chart: all templates (`deployment.yaml`, `service.yaml`, `route.yaml`)

RHDP uses this label to:
- Monitor deployment health
- Track resource usage
- Report status back to the catalog
- Enable lifecycle management
